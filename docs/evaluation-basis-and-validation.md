# InkPi 评价依据与验证计划

## 1. 项目定位

InkPi 当前不是“自动给书法考级结果”的系统，而是一套面向：

- 毛笔楷书单字练习
- 毛笔行书单字练习
- 初学者与课堂即时反馈
- 设备端正式展示与云端复盘

的 **辅助评测系统**。

更稳妥的表述是：

- `total_score` 是当前统一主分
- 新的五维 `rubric` 是 **来源明确的正式评审标准层**
- 当前正式支持楷书与行书单字
- 其他书体与多字作品不在当前正式范围内
- 系统服务于辅助练习，不替代教师终评和正式考级

## 2. 标准来源目录

InkPi 的新标准只使用公开可查、且能落到当前产品场景的来源。

| 来源码 | 来源 | 机构 | 作用 |
| --- | --- | --- | --- |
| `MOE-2013` | 《中小学书法教育指导纲要》 | 教育部 | 给出“规范书写、技能训练、评价多元”的教育边界 |
| `MOE-2025` | 对十四届全国人大二次会议第 4162 号建议的答复 | 教育部 | 明确 2022 课标口径下的 progression：楷书到规范、通行的行楷/行书 |
| `SC-SHFA-2018` | 四川省书法水平测试毛笔书法测试大纲 | 四川省教育考试院 | 给出最适合初学者场景的权重骨架：笔法、字法、章法、卷面 |
| `CAA-EXAM-2018` | 全国美术考级软笔书法考试与培训标准 | 中国美术学院社会美术水平考级中心 | 补强提按顿挫、线条力度、结构准确度、章法完善等描述 |
| `CWA-REVIEW-2024` | 全国第十三届书法篆刻展评审评议 | 中国书法家协会相关评审语境 | 提供笔法、结构、章法、墨法、笔力、规范识别等正式评审话语 |

这些来源在系统中的作用不是“照搬考级规则”，而是：

1. 给出一套 **有出处的维度语言**。
2. 给出适合单字练习的 **权重骨架**。
3. 给出未来重训与人工校核时可复用的 **统一口径**。

### 3.2 单字适配说明

比赛和协会语境中的“章法”常指整幅作品的布局关系。InkPi 当前只做单字，因此把它适配为：

- 字内布白
- 主体位置
- 重心与四边留白
- 单字内的疏密关系

这不是把整幅作品章法强行缩小，而是把“章法意识”落到单字产品能稳定观测的范围里。

### 4.2 为什么行书与楷书必须分开

楷书和行书不能共用一套维度和一套解释文案，原因很直接：

- 楷书更强调规范、结构、起收笔与稳定边界
- 行书更强调连带、节奏、流动性下的完整与可识别

因此 InkPi 在运行时不是“一个 rubric 打所有字”，而是：

- 用户先选 `script`
- OCR 后按 `script` 路由主分模型
- rubric engine 再按 `script` 生成对应五维结果

## 5. 评分机制与运行时映射

### 5.1 当前机制

- 每个维度输出 `0-100`
- 当前锚点固定为：`20 / 40 / 60 / 80 / 100`
- 每档都绑定来源化描述
- 系统内部计算 `rubric_preview_total`
- 过渡期不对外展示 `rubric_preview_total`

### 5.2 与现有模型的关系

当前采用“两层评分”：

1. **现有 ONNX 主分层**
   - 继续输出 `total_score`
   - 继续输出 `quality_level`
   - 继续提供概率、置信度、质量特征和校准中间量

2. **新的 rubric engine**
   - 使用 OCR、几何特征、质量特征和校准中间量
   - 生成五维正式评审项
   - 生成 `rubric_preview_total`
   - 生成基于来源的依据说明

当前阶段最重要的约束是：

- **不改正式 `total_score`**
- **先替换维度层，再准备重训**

### 5.3 现有特征如何服务新标准

当前系统并未重写 OCR 和预处理，而是把已有信号重新组织到新 rubric 中：

- 结构相关：`center_quality`、`bbox_ratio`、`projection_balance`、`bbox_fill`
- 笔法 / 线质相关：`texture_std`、`orientation_concentration`、`ink_ratio`
- 规范 / 识别相关：`ocr_confidence`、`dominant_share`、`edge_touch`
- 墨法 / 笔力相关：墨量、纹理稳定性、连贯性与 ONNX 中间量
- 稳定辅助：`quality_confidence_norm`、`probability_margin_norm`、`score_range_fit`

这意味着当前标准重构是 **方法论层重构**，不是重新发明第二套输入链路。

## 6. 结果结构与迁移策略

当前标准结果对象新增或替换这些公共字段：

- `rubric_version`
- `rubric_family`
- `rubric_items`
- `rubric_summary`
- `rubric_source_refs`
- `rubric_preview_total`

迁移策略固定为：

- 新记录：写 `rubric_*`
- 旧记录：标记为 `legacy_v0`
- 不把旧四维强行回填为新标准
- 新 UI 只对新记录显示正式 rubric
- 旧记录显示“旧版评测标准记录”

## 7. 当前验证路线

为了回应“数据不够”和“评价依据不够权威”的问题，InkPi 当前采用以下验证路线：

1. 先保持 `total_score` 不变，避免新标准上线同时引入主分波动。
2. 让新五维直接上屏，验证它们在 Qt、小程序、Cloud API、运维后台上的表达是否一致。
3. 累积 `rubric_items` 与人工复评记录，形成后续重训所需标签。
4. 通过统计页和验证总览接口持续观察：
   - 样本量
   - 覆盖汉字数
   - 书体分布
   - 设备来源
   - 专家复评覆盖率
   - AI 与人工一致率
   - AI 与人工平均分差

建议后续阶段目标：

- `500+` 份带标签样本
- `3+` 名教师或专家参与校核
- `20+` 名试用用户形成连续练习记录

## 8. 当前不纳入评分的内容

为了避免“看起来更完整，实际上没有依据”的扩张，以下内容 **不纳入当前单字系统评分**：

- 整幅作品章法
- 文辞内容
- 落款与印章
- 多字通篇气息
- 草书、篆书、隶书的跨书体混评

这不是忽略这些因素，而是承认当前产品边界。

## 9. 产品闭环意义

这次标准重构不是把名字改漂亮，而是让整套产品闭环更稳：

- Qt：结果页与历史页显示新五维正式评审项
- 小程序：详情页、统计页、方法论页显示来源化依据
- Cloud API：返回 `rubric_definitions`、`rubric_source_catalog`、`current_script_scope`
- 运维后台：明确区分当前主分与新的 rubric 家族
- 训练链路：先准备 manifest 和人工评分对照字段

这样项目对评委、老师和用户都能说清楚三件事：

1. 我们现在按什么标准评。
2. 这个标准来自哪里。
3. 为什么当前先不切正式主分。
