# InkPi 评价依据与验证计划

## 1. 当前项目定位

InkPi 当前不是“自动给书法考级结果”的系统，而是一个面向 **楷书 + 行书单字练习** 的辅助评测系统。

更适合的场景：

- 初学者单字练习
- 课堂展示或课程实验
- 设备端即时反馈
- 教师辅助回看与筛选样本

当前边界：

- `total_score` 是统一主分
- 四维分是解释层，不直接替代教师终评
- 当前正式支持 `楷书单字` 与 `行书单字`
- 其他书体与多字作品当前不支持

## 2. 四维评价依据

四维名称不是随意命名，而是把书法课堂中的常见观察维度映射到可计算特征。

| 维度 | 回答的问题 | 课堂观察点 | 对应特征 |
| --- | --- | --- | --- |
| 结构 | 字形重心、比例和空间分布是否稳定 | 重心、留白、宫格比例、主部件位置 | `center_quality` `bbox_ratio` `projection_balance` `bbox_fill` |
| 笔画 | 笔画粗细、走向和控制是否稳定 | 粗细波动、方向、边缘、连带与转折 | `texture_std` `orientation_concentration` `ink_ratio` `component_norm` |
| 完整 | 单字主体是否完整、清晰、可识别 | 缺笔、断裂、粘连、碰边、噪声 | `ocr_confidence` `dominant_share` `edge_touch` `subject_edge_safe` |
| 稳定 | 本次评测结果是否稳定、可信 | 主分区间、概率集中度、重复评测波动 | `quality_confidence_norm` `probability_margin_norm` `feature_quality` `score_range_fit` |

## 3. 双书体方法论

当前系统对两种正式支持书体采用不同解释口径：

- `楷书`：强调结构规范、重心、比例、起收笔稳定
- `行书`：强调连带、节奏、流动性下的结构完整与稳定识别

因此四维名称保持不变，但每一维的说明、练习建议和边界声明都按 `script` 输出。

## 4. 量化证据

系统当前可自动沉淀以下量化信息：

- 总样本数
- 书体分布
- 覆盖汉字数
- 设备来源数
- 最近 7 天样本数
- 分数分布
- 四维均值
- 高频练习字
- 设备贡献度
- 专家复评覆盖率
- AI 与人工一致率
- AI 与人工平均分差

这些指标已经进入云端接口与小程序统计页，可直接用于答辩展示。

## 5. 验证计划

为了回应“数据依据不够”和“评价标准不够权威”的问题，当前版本采用以下验证路径：

- 保留人工评分对照表，不把四维分包装成官方等级结论
- 先验证主分与人工判断的一致性，再逐步校准四维解释
- 记录设备来源、书体分布、时间分布和高频字，用于样本分析
- 为每条结果补充教师 / 专家复评记录，形成可追溯验证快照

建议的后续目标：

- `500` 份以上带标签样本
- `3` 名以上教师 / 专家参与校核
- `20` 名以上试用用户形成连续练习记录

## 6. 产品化闭环

为了回应“不能只告诉用户写得不好，还要给出指导”的意见，当前版本已经补上：

- 小程序统计页：量化证据、验证计划、评价依据来源
- 小程序详情页：四维依据卡、当前阶段、优先提升项、练习动作建议
- 历史管理：单条删除、批量删除、筛选与记录整理
- 云端复评接口：支持教师 / 专家提交人工评分与维度评语
- 验证总览接口：直接返回复评覆盖率、一致率与平均分差

这意味着项目已经开始从“评测 demo”转向“可解释、可量化、可持续优化”的教学型产品。
